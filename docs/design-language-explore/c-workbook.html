execution / runtime

Notes on how a turn starts, where the mode decides, what the host satisfies, and what the projector hands back to the model. Open on the bench, not the lectern.

04.2 · mode dispatch M.01 · standard M.02 · rlm ↳ ModePreset ↳ RlmConfig

Mode execution

premise. Two modes ship as ModePresets — standard() and rlm()1. Each contributes a protocol driver and preamble, so the runtime loop does not need provider-specific branches. Background work and handoffs are cross-cutting outcomes either mode can coordinate with, not modes of their own.

M.01 std

native provider tool calls

Standard mode interprets provider output through StandardDriver and emits DriverAction::StartTools. The runtime dispatches mode-native tools first, then registered ToolProviders.

M.02 rlm

persistent lashlang execution

RLM mode extracts lashlang fences and runs them through a persistent VM. Read-only host bindings project into Lashlang scope via ProjectedBindings. continue_as, llm_query, and the stream_mask plugin manage handoff, completion, and live UI masking.

noteHandoffs are an outcome, not a mode. An RLM block ending with continue_as { task, seed } finishes the low-level turn as TurnOutcome::Handoff; the facade follows it and returns the final TurnResult.

04.3 · facade ↳ LashCore ↳ LashSession ↳ TurnBuilder ↳ TurnActivitySink

Core shape

facade. A LashCore is cloneable shared configuration (provider, model, modes, plugins, store factory). A LashSession wraps a parked, resumable runtime for one conversation. A TurnBuilder drives a single turn — synchronously through .run() or via a sink with .stream(&sink).

examples/hello-turn.rsrust
// hello, turn
let core = LashCore::standard()
    .provider(provider)
    .model("anthropic/claude-sonnet-4.6", None)
    .max_context_tokens(200_000)
    .store_factory(store_factory)
    .build()?;

let session = core.session(chat_id).standard().open().await?;
let result = session
    .turn(TurnInput::text(user_text))
    .stream(&events)
    .await?;

04.6 · principles § 1 — accent § 2 — structure § 3 — editorial § 4 — calm

Principles

  1. Accent means action

    Sodium is reserved for the active edge — current section, plan mode, prompts. Passive surfaces use chalk hierarchy instead. The accent works because it is rare.

  2. Structure before ornament

    Every rule, rail, and block should clarify hierarchy before it tries to look good. Beauty comes from discipline; if a divider is decorative, remove it.

  3. Editorial breaks the grid

    Three card grids in a row is a template, not a design. At least one section per page uses a non-card layout — hanging numbers, hairline-divided lists, asymmetric prose.

  4. Long sessions stay calm

    No high-saturation backgrounds, no fragile thin text, no ornamental overload. The palette and typography survive hours of reading.
