$ section 04.2 mode-execution

mode execution

Two modes ship as ModePresets — standard() and rlm(). Each contributes a protocol driver and preamble, so the runtime loop does not need provider-specific branches. Background work and handoffs are cross-cutting outcomes either mode can coordinate with, not modes of their own.

$lash modes --list
M.01 standard interprets provider output through StandardDriver and emits DriverAction::StartTools. Runtime dispatches mode-native tools first, then registered ToolProviders.
M.02 rlm extracts lashlang fences and runs them through a persistent VM. Read-only host bindings project via ProjectedBindings. continue_as, llm_query, and stream_mask manage handoff, completion, and UI masking.

$ section 04.6 principles

principles

  1. $rule 01accent-means-action

    accent means action

    Sodium is reserved for the active edge — the cursor, the current section, the prompt sigil. Passive surfaces use ink hierarchy instead. The accent works because it is rare.

  2. $rule 02structure-before-ornament

    structure before ornament

    Every rule, rail, and block clarifies hierarchy before it tries to look good. The chrome speaks shell because lash is a shell — not because shell looks cool.

  3. $rule 03editorial-breaks-the-grid

    editorial breaks the grid

    Three card grids in a row is a template, not a design. Each chapter holds at least one section that breaks the shell idiom — a typeset table, a pull quote, a body paragraph that simply runs.

  4. $rule 04long-sessions-stay-calm

    long sessions stay calm

    No CRT scanlines, no Matrix green, no chromed neon. The terminal here is a typographic device, not a costume. Warm paper, slow cursor, restrained accent.
