Execution starts in the CLI bootstrap, then branches into interactive TUI or the autonomous --print path. Inside a turn, the active mode decides whether model output becomes native tool calls or Lashlang code execution.

§ 04.2 · mode dispatch

mode execution

M.01–M.02 
═══════════════════════════════════════════════════════════════════════════════

Two modes ship as ModePresets — standard() and rlm(). Each contributes a protocol driver and preamble, so the runtime loop does not need provider-specific branches. Background work and handoffs are cross-cutting outcomes that either mode can coordinate with, not modes of their own.

╭─[M.01]──────────────────────╮
│            standard         │
╰─────────────────────────────╯

native provider tool calls

Standard mode interprets provider output through StandardDriver and emits DriverAction::StartTools. The runtime dispatches mode-native tools first, then registered ToolProviders.

╭─[M.02]──────────────────────╮
│            rlm              │
╰─────────────────────────────╯

persistent lashlang execution

RLM mode extracts closed lashlang fences and runs them through a persistent VM. Read-only host bindings project via ProjectedBindings. continue_as, llm_query, and stream_mask manage handoff, completion, and UI masking.

A turn that ends with continue_as becomes a handoff, not a new mode:

§ 04.3 · facade

core shape

api · 3 types 
───────────────────────────────────────────────────────────────────────────────

The facade is small. A LashCore is shared configuration; a LashSession is one conversation; a TurnBuilder drives a single turn through a sink.

examples/hello-turn.rsrust · 11 lines
// hello, turn
let core = LashCore::standard()
    .provider(provider)
    .model("anthropic/claude-sonnet-4.6", None)
    .max_context_tokens(200_000)
    .store_factory(store_factory)
    .build()?;

let session = core.session(chat_id).standard().open().await?;
let result = session
    .turn(TurnInput::text(user_text))
    .stream(&events)
    .await?;

§ 04.6 · principles

principles

4 rules 
───────────────────────────────────────────────────────────────────────────────

  1. accent means action

    Sodium is reserved for the active edge — current section, plan mode, prompts. Passive surfaces use ink hierarchy instead. The accent works because it is rare.

  2. structure before ornament

    Every rule, rail, and block should clarify hierarchy before it tries to look good. Beauty comes from discipline; if a divider is decorative, remove it.

  3. editorial breaks the grid

    Three card grids in a row is a template, not a design. At least one section per page uses a non-card layout — hanging numbers, hairline-divided lists, asymmetric prose.

  4. long sessions stay calm

    No high-saturation backgrounds, no fragile thin text, no ornamental overload. The palette and typography survive hours of reading.
