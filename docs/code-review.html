Top-line verdict

The architecture is healthy and the just-landed work was real. Storage's with_write_tx/guard_lease, the anthropic/google provider decomposition, the unified RLM fence_scan, the truncate_windowed core, the CheckpointEffect deletion, and the facade's PromptLayerSink all landed cleanly. No structural regression was introduced.

But the dominant finding is a consistent half-finished-cutover pattern. Each dedup created the canonical helper and pointed some callers at it — then stopped. The superseded copy survives next to the new one as a second authority that can silently drift: byte-identical, hand-synced, and (in three cases) already diverging. This is the highest-leverage cleanup available, and it is pure subtraction.

Secondary: a cluster of parallel walkers / forwarder strata (four node-classifiers, two graph walkers, a 16× result-envelope, a 38-method pass-through layer, a 5-crate plugin shim) and a file-size signal that is half-illusory — most “1000+ line” files are test-inflated; only seven are genuinely large code.

Top 10 code-judo moves

Ordered by deletion-to-risk: lines removed plus drift-risk eliminated, all behavior-preserving. CODE-JUDO DEAD CODE

  1. ~250 lines

    Replace the 16× hand-spelled grep result envelope with one GrepEnvelope struct

    The 14–16-key result map is re-typed at 16 exit paths (10 are the empty/error shape), with a 17th hand-synced copy in the output schema. One branch already omits two fields. This is the first review's "GrepEnvelope ×11", undercounted and unaddressed.

    Fix: #[derive(Serialize)] struct GrepEnvelope + GrepError + Default; builders become ..Default::default(); schema derives.
    lash-tool-search/src/lib.rs (16 sites; 222–1523)
  2. ~250 lines

    Inline the SessionControl pass-through stratum into the namespacing structs

    ~38 private SessionControl methods exist only to be re-forwarded 1:1 by the 11 *Control structs that already are the public surface — three stacked forwarding layers for one op.

    Fix: move each body into its single caller; keep only with_writer/tool_state shared. Deletes one whole layer, zero API change.
    lash/src/control.rs:60–557
  3. ~250 lines

    Collapse the 5-crate Factory→Plugin shim onto StaticPluginFactory + a configured_plugin_factory! macro

    rolling-history, tool-output-budget, observational-memory, standard-protocol (and MCP) each hand-roll Plugin+register()+Factory{id,build} that PluginSpec already expresses; the newtype delegation is itself copy-pasted in the crates that "do it right".

    Fix: build a PluginSpec in the ctor + store a StaticPluginFactory; a macro generates the newtype+delegation. (Stateful plan-mode/update_plan/rlm correctly excluded.)
    5 plugin crates + new macro in lash-core/plugin/registry.rs
  4. ~200 lines

    Delete the RLM history renderer's #[cfg(test)] shadow path

    ~200 lines of driver/history.rs are a second, owned-data rendering implementation kept in parallel with the production borrowed path; tests validate the shadow, not production.

    Fix: drive the production borrowed functions from real fixtures (or a ~5-line owned→borrowed adapter). Drops the file under 600.
    lash-protocol-rlm/src/driver/history.rs
  5. ~170 lines

    Emit SurfaceRequirements from lowering; delete RequirementsCollector

    The linker's lowering walk already computes every input the surface-requirements walker re-derives (resource bindings, op host-names, ability flags). The second engine is a drift risk on the surface the linker just proved consistent.

    Fix: make the lowering pass emit requirements as a byproduct; delete collect_expr + the duplicate RequirementBinding enum + scope logic.
    lashlang/src/artifact.rs:945–1283
  6. ~115 lines

    Wire OpenAI/Codex onto transport::normalize; delete the dead lift

    frame_sse_payload has no callers while OpenAI keeps two byte-identical framers; merge_usage/terminal_reason_from_parts/serialize_options_tail are shadowed locally. The "single canonical impl" is false for the two highest-volume providers.

    Fix: make the OpenAI framers two-line shims over frame_sse_payload; delete local merge_usage/terminal_reason_from_parts and call serialize_options_tail.
    lash-provider-openai/src/{responses_shared,chat,provider}.rs · lash-provider-codex/src/lib.rs
  7. ~110 lines

    Delete the dead transcript renderer

    render_transcript_prompt + render_message_for_transcript + TranscriptTurn have zero non-test callers — kept alive only by their own tests (which is why cargo/spring-cleaning didn't flag them). Active path is append_structured_prompt.

    Fix: delete the three items, both re-exports, and the tests that exercise only them. Pure subtraction.
    lash-sansio/src/session_model/message.rs:233–254, 519–522, 587–660
  8. ~100 lines

    Collapse the four session-graph node-classifier walkers into one classifier + sink

    The "push event; push message if non-transient&unseen; else push tool_call" cascade is hand-inlined in four places (the first review's "3 parallel walkers" is four). Plus a unique_*_node_id vs _for_replacement quartet that differs only by a second dedup set.

    Fix: one classify_node(node, &mut impl ReadModelSink); thread a single owned id set so the _for_replacement twins delete.
    lash-core/src/session_graph.rs:441–535, 1304–1449
  9. drift fix

    Move the builtin name→op mapping onto the registry; unify the three approx_token_count copies

    Two "single sources" that are really two: BUILTINS + intrinsic_for_builtin. Separately, "≈ len ÷ 4" is copied across three plugins that share a context budget — and OM counts chars while the others count bytes.

    Fix: add an op to each Builtin; one approx_token_count in lash-core consumed by all three plugins.
    lashlang/src/{builtins,runtime/compiler}.rs · 3 plugin crates
  10. ~80 lines

    Consolidate the rolling-history ↔ tool-output-budget spill fork; shrink the dead typestate

    The truncation core was unified but disk-spill, direction policy, and token estimate stayed forked (already differ on the hash key). Separately, turn_commit_pipeline::final_commit has an unreachable Drafting branch + dead Option/None arms (stage is always Finalized by then).

    Fix: move spill/direction into tool-output-budget's public surface; replace the stage enum's panic-guarded accessors with a single finalized_progress_graph().
    lash-plugin-rolling-history/src/lib.rs:129–236 · lash-core/src/runtime/turn_commit_pipeline.rs:411–447

Scope & method

How this was run

Six independent read-only reviewers, one per subsystem (core+sansio · lashlang · providers+transport · tools+stores · plugins+protocols+subagents · facade+ui+trace), each applying the thermo-nuclear lens (code-judo simplification, >1k-line decomposition, spaghetti/branching, duplication, abstraction/boundary quality) to whole crates, not a diff. Every crate except lash-cli was covered. Nothing was edited.

Findings are structural-quality analysis. Line-deletion estimates are reviewer approximations. The code vs test split in §D was measured per file (#[cfg(test)] boundary), correcting the raw line-count signal.
