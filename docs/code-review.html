Top-line verdict

Architecture is sound. There is no structural regression anywhere. The pervasive structural debt is horizontal duplication — the same algorithm, scaffold, CAS-invariant, builder-surface, or string-primitive written 2–4× by hand, with several copies already drifting.

The single recurring theme across every crate: a canonical helper exists or obviously should — and copies proliferated instead.

Separately, the bug-hunt found real P1 correctness bugs concentrated at two seams: out-of-band token-usage accounting (counted twice on replay, and committed outside the lease fence) and byte-level provider stream framing (UTF-8 corruption, dropped terminal events). One storage finding rises to P0: the durability CAS/lease guards are enforced in Rust around the SQL, not atomically by it.

Lens A — Structural · complete coverage
A Required restructurings

The top 5, ordered by leverage

Each is a code-judo move — it deletes complexity rather than rearranging it. Ordered by deletion-to-risk ratio. STRUCTURAL · REQUIRED

  1. Delete the parallel enum CheckpointEffect

    Two enums, the same 10 variants, ~150 lines of from_effect/into_effect round-tripping — differing only in MessageSequence vs Vec<Message> for serde.

    Fix: make Effect serde-capable, delete the twin + both converters. Highest deletion-to-risk ratio in the workspace.
    lash-sansio/src/sansio/mod.rs:180-288, 683-771

  2. Add an Expr traversal primitive in lashlang & fold validate into lower

    ~12 hand-written full-AST walks. validate_expr/lower_expr are near-identical twins (the tree is walked twice); artifact.rs re-implements the linker's scope/binding inference a third time; the builtin table exists 3×; type→schema construction 3–4×.

    Fix: Expr::children(); emit SurfaceRequirements as a lowering byproduct; one builtin registry; one fold_type authority.
    lashlang/src/{ast.rs, linker.rs, compiler.rs, artifact.rs}

  3. with_write_tx/with_read helper + one guarded-fenced-lease predicate

    The lock+txn+commit+error ceremony is open-coded in ~16 methods. The lease-fencing CAS — the most safety-critical SQL in the crate — is duplicated near-verbatim between turn-leases and process-leases on two error channels (a code comment literally says "mirrors claim_runtime_turn_lease").

    Fix: two helpers collapse the ceremony and make the _conn-vs-&self twins trivial wrappers; extract one guard_lease predicate shared by both lease types.
    lash-sqlite-store/src/{persistence.rs, process_registry.rs, graph.rs, blobs.rs}

  4. Decompose the two 1500-line provider monoliths + promote 5 helpers to transport

    OpenAI already demonstrates the split (config/request/stream/policy/factory); Anthropic (1535) and Google (1595) are flat monoliths. merge_usage, terminal_reason_from_parts, SSE-payload framing, the HTTP-error envelope, and the serialize_config options-tail each exist 2–4×, several stranded in the OpenAI crate where peers can't reach them.

    Fix: split both files along OpenAI's seams; lift the 5 primitives into lash-llm-transport. A new normalizer trait is not warranted — the conformance suite already pins the output contract.
    lash-provider-anthropic/src/lib.rs (1535), lash-provider-google/src/lib.rs (1595)

  5. Unify the two tool-output truncation + disk-spill implementations

    The same head/tail-window + byte-cap + spill algorithm written line-for-line in two plugins. Rolling-history imports the budget plugin's constants, then forks the logic — and the tables have already diverged (rolling-history tails "batch", the budget doesn't).

    Fix: one canonical truncate_windowed + spill helper; rolling-history consumes the budget projector instead of re-formatting.
    lash-plugin-rolling-history/src/lib.rs:139-264 · lash-plugin-tool-output-budget/src/lib.rs:321-527
