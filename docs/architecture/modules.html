Crate Families

These are the high-value ownership groups to inspect before changing behavior.

Pure protocol

lash-sansio

Owns TurnMachine, Effect, Response, ModePreamble, prompt templates, messages, attachments, and tool definitions/output contracts.

lash-sansio/src/{sansio,turn,mode,prompt,tool_surface}.rs
Async runtime

lash

Owns LashRuntime, Session, SessionGraph, and the RuntimePersistence contract, plus tool dispatch, provider registry, MCP, OAuth, attachments, tracing, and runtime controls.

lash/src/runtime/*, lash/src/session.rs, lash/src/store.rs
Embed API

lash-embed

Provides app-facing runtime construction, managed chat sessions, parking/resume, and webserver embedding without pushing CLI or TUI vocabulary into core runtime.

lash-embed/src/lib.rs, examples/*
Mode plugins

standard + rlm

Standard drives native provider tool calls. RLM extracts lashlang code fences, runs persistent REPL state, snapshots execution globals, and records RLM trajectory.

lash-mode-standard, lash-mode-rlm, lashlang
Provider plugins

OpenAI, Codex, Anthropic, Google

Each provider implements state, auth, readiness, transport, and model policy components behind ProviderHandle.

lash-provider-*, lash-providers-builtin
Tool plugins

default tools + subagents + LLM helpers

Default tools, subagents, llm_query, monitors, plan mode, prompt context, observational memory, UI activity, and tool discovery register through plugin factories. Discovery remains split across catalog projection, provider execution, lexical ranking, LLM rerank, and schema indexing.

lash-default-tools, lash-subagents, lash-llm-tools, lash-plugin-*
Persistence

lash-sqlite-store

The concrete SQLite implementation persists session heads, graph nodes, blobs, checkpoints, usage ledgers, tombstones, and vacuum metadata behind the runtime store contract.

lash-sqlite-store/src/lib.rs
Host UI

CLI, TUI, UI surfaces

The CLI bootstraps runtime, plugins, and providers, then renders the authoritative SessionReadView through ChronologicalProjection into UiTimeline.

lash-cli, lash-ui, lash-tui
Inspection tools

trace + export + file index

lash-trace defines standard and extended JSONL trace records. lash-export combines store state with provider traces to render chronological sessions, prompt snapshots, and token usage. lash-file-index powers async fuzzy file completion.

lash-trace, lash-trace-viewer, lash-export, lash-file-index
Experimentation

benchmarks + GEPA + autoresearch

Benchmark runners embed Lash as an evaluated agent. GEPA owns optimizer primitives, while Autoresearch adds hidden research tools and UI extension surfaces.

bench/*/runner, lash-gepa, lash-autoresearch
