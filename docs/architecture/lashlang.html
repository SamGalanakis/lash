Language Surface

The prompt-facing language is intentionally Python-shaped while staying typed enough for tool contracts and structured output.

Values

null, booleans, numbers, strings, lists, records, results, async handles, and immutable Image handles. Images expose .id, .label, .size, .width, and .height.

Control flow

Assignments, functions, loops, conditionals, ? fail-fast unwrapping, parallel, start call, await, and cancel.

Tools

call tool { ... } returns a result wrapper; (call tool { ... })? unwraps success or stops the program with the tool error.

Output

print sends observations to the next model call. submit ends the turn with final text or JSON-compatible metadata.

RLM Execution Path

The model emits code, the runtime executes it locally, and the resulting trajectory is persisted as mode events.

Images And Attachments

Image support is a first-class runtime feature, not a plain record convention.

print(image)

Collects image values recursively, registers their attachment refs, and sends provider-visible LlmContentBlock::Image content alongside descriptor text.

submit(image)

Serializes metadata only, using a JSON-compatible descriptor with id, label, size, width, and height. It does not embed base64 data.
