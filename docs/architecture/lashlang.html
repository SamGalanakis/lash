Core Syntax

The language reference is shared by RLM and RLMPure. Keep docs here aligned with LASHLANG_LANGUAGE_REFERENCE.

Values And State

  • Values: null, booleans, numbers, strings, lists, and records.
  • Literals: [a, b] and { a: 1, b: 2 }.
  • Assignment: name = expr, plus variable-rooted path updates like record.field = value, record[key] = value, list[i] = value, and state.groups[g].count = count + 1.
  • Record assignment inserts or replaces fields. Dynamic record indexing reads string-coerced keys and returns null when absent, so counts[g] = counts[g] + 1 is a compact histogram pattern.
  • List assignment replaces an existing integer index, including negative indexes; it does not grow the list.
  • Variables persist across fenced blocks within the same turn.
  • Bare expressions are valid statements. Inside parallel { ... }, a bare expression contributes to the positional result list.

Control Flow

  • Statement control flow: if and for.
  • break exits the nearest for; continue skips to that loop's next iteration.
  • Expression conditional: cond ? yes : no.
  • There is no expression-form if.
  • Boolean negation: !cond or not cond.
  • submit is not loop control. It exits the whole program and delivers the turn's final value.
  • If a prompt includes bound variables, those names are already in scope.

Tool Calls

Every mode-owned tool listed under Available Tools is callable from lashlang, even though RLM and RLMPure send no native provider tool schema.

Form Meaning Result shape
call tool { arg: expr } Run a tool synchronously. { ok: true, value: ... } or { ok: false, error: "..." }
(call tool { ... })? Fail-fast unwrap for the common happy path. Returns .value or aborts the block with the tool error.
start call tool { ... } Start a background call. A handle. Resolve with await handle.
await [h1, h2] Wait for multiple handles. A list of wrapper records in order.
cancel handle Best-effort cancellation. Cancellation status from the runtime.
parallel { a: call ... b: call ... } Run independent branches concurrently. A record for named branches, or a list for positional branches. 
fanout = parallel {
  cargo: call exec_command { cmd: "cargo check --workspace" }
  fmt: call exec_command { cmd: "cargo fmt --check" }
}
print { cargo_ok: fanout.cargo.ok, fmt_ok: fanout.fmt.ok }

Type Literals

Type literals describe expected shapes for validation and typed RLM outputs.

Shapes

  • Record type: Type { field: shape, ... }.
  • Scalars: str, int, float, bool, dict, any, null.
  • Collections: list[shape], enum["a", "b"], nested Type { ... }.
  • Unions: a | b | c, for example status: str | int.

Absent Versus Null

  • Optional field: email: str? means the field may be absent. If present, it must be a string.
  • Nullable field: email: str | null means the field is required and may be string or null.
  • Nested record types require Type. Bare { ok: bool } is a record value, not a type.
candidate = { title: "Architecture", status: "draft", tags: ["docs"] }
validated = validate(candidate, Type {
  title: str,
  status: enum["draft", "ready"],
  tags: list[str],
  reviewer: str?,
  published_at: str | null,
})

Protocol Rules

These rules are what keep RLM/RLMPure chronological and inspectable in the UI.

One block
Only the first accepted fenced lashlang block executes.Trailing prose and later fences do not drive execution. Prompts tell the model to emit exactly one block.
Observe first
print before acting on uncertain outputs.The next iteration receives the observation in trajectory history.
Submit last
Do not submit in the same block that gathers fresh evidence.Run the block, inspect the observation, then submit in a later block if ready.
Tool results
Tool events are recorded before the trajectory entry.The timeline can render thinking, tool results, observations, and final response in causal order.
