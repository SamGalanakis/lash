Core Abstractions

The runtime stays flexible because mode behavior is data and traits, not special-cased branches in the provider loop.

Abstraction Current role
ExecutionModeUser-facing selection for standard, rlm, and rlmpure.
PromptTemplateOrdered sections and slots used to construct the system prompt.
PromptContributionMode, plugin, project, runtime, and environment prompt fragments merged into template slots.
ModePreambleMode-owned protocol, projector, tool specs, tool names, execution prompt, and prompt contributions.
ContextProjectorTurns prepared state into an LlmRequest. Standard/RLM use chat projection; RLMPure uses explicit inputs plus REPL history.
ProtocolDriverHandleInterprets provider responses and tool results into declarative driver actions.
TurnMachineSans-io state machine that alternates model calls, effects, retries, checkpoints, and terminal responses.
RuntimeCoreConfigRuntime-level configuration used to wire providers, tools, modes, and persistence.
EmbeddedRuntimeBuilderEmbeddable construction API for host applications and tests.
PluginFactory / SessionPluginPlugin lifecycle for tools, prompts, UI activity, storage, and session hooks.
ProviderFactory / ProviderHandleProvider registration and streaming/completion boundary.
RuntimePersistenceComposite runtime persistence capability over blob, head, graph, usage, and retention stores.
SessionGraphDurable event graph used for replay, projection, ordering, and TUI rendering.
RlmTrajectoryEntryPersistent record of one lashlang iteration: reasoning, code, output, observations, tool calls, errors, and final output.

Maintenance Rules

These are the rules this documentation expects the codebase to preserve.

Mode parity
All modes should render chronological events.Thinking, tool calls, tool results, lashlang observations, and final assistant output should appear in causal order.
Prompt reuse
Lashlang docs should stay single-sourced.RLM and RLMPure both reuse the shared language reference; documentation should mirror that shared text instead of inventing mode-specific variants.
Provider contract
Native tool schemas belong only to Standard mode.RLM and RLMPure document tools in prompt text and execute through lashlang.
Event order
Record tool results before RLM trajectory entries.This keeps replay and UI projection aligned with what happened during execution.
