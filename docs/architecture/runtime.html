Major Modules

Paths are grouped by ownership, not by build order.

Pure protocol

lash-sansio

Owns TurnMachine, ModeConfig, prompt templates, message/session model types, tool definitions, and typed turn effects.

Async runtime

lash/src/runtime

lifecycle.rs builds/resumes runtime, session_api.rs exposes host APIs, turn_loop.rs finalizes turns, and turn_driver/ performs provider/tool/code effects.

Session runtime

lash/src/session.rs

Maintains per-session tool surface, plugin session, lashlang runtime, globals patches, execution surface sync, and dynamic tool visibility.

Durable timeline

lash/src/session_graph.rs

Persists conversation events, tool invocation events, mode events such as RLM trajectory, state snapshots, plugin nodes, and active path metadata.

Plugin API

lash/src/plugin

Mode protocol drivers, mode session plugins, native tools, prompt hooks, context surface, child session creation, and runtime extension points.

CLI host

lash-cli

Bootstrap, provider setup, TUI state, rendering, slash commands, resume/fork, session logs, and host-managed docs.
