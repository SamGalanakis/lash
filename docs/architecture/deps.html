Provider Details

Provider crates share the same runtime interface but own their own auth, readiness, model policy, and request transport.

ProviderKindTransport / Auth
OpenAI APIopenaiUses bearer API-key auth against https://api.openai.com/v1/responses. Direct OpenAI is Responses-only and owns Responses reasoning replay plus prompt cache fields.
OpenAI-compatibleopenai-compatibleUses bearer API-key auth against a caller-provided base URL and posts Chat Completions requests to {base_url}/chat/completions.
Codex subscriptioncodexPosts to ChatGPT Codex Responses backend with OAuth device-code tokens and Codex-specific headers.
AnthropicanthropicPosts to /v1/messages with API key, Anthropic version header, and beta flags.
Google Gemini / Code Assistgoogle_oauthUses Google OAuth PKCE/manual-code flow and Code Assist generate/stream endpoints.
OpenAI provider split
openai is the direct OpenAI Responses provider and accepts no base_url. openai-compatible is the generic Chat Completions provider and requires a caller-provided base_url.
Responses path
The direct OpenAI path posts to https://api.openai.com/v1/responses, keeps Responses reasoning replay, and uses prompt_cache_key from the Lash session id when cache retention is enabled. Long retention adds prompt_cache_retention where that API supports it.
Chat Completions path
The Chat path posts to {base_url}/chat/completions, converts LlmRequest into messages, emits Chat Completions tools, maps structured output to response_format, and preserves OpenRouter reasoning effort through reasoning.effort.
Claude cache markers
For OpenRouter Claude on Chat Completions, cache_retention controls Anthropic-style cache_control markers. short emits {"type":"ephemeral"}; long adds "ttl":"1h"; none emits no markers. Breakpoints are the first system/developer text message, the last tool definition, and any explicit LlmContentBlock::Text.cache_breakpoint; without an explicit block, the provider falls back to the last user/assistant text content.
RLM prompt caching
RLM projects chronological history as append-only chat-shaped messages: user inputs remain user messages, prior RLM steps become assistant messages, tool observations become user messages, and the mutable current-iteration/finalization prompt stays as the final user message. The rolling cache_breakpoint is placed on the last stable history text block, so OpenRouter Claude can cache a real message prefix instead of a rewritten history blob.
Usage and cost inputs
Provider responses still return normalized LlmUsage to the runtime usage ledger. Chat parsing reads streaming and non-streaming usage chunks, including OpenRouter cache fields; cache-write tokens are not counted as cached reads, so downstream cost/export code does not overstate cache hits.
Provider boundary rule
Wire-specific schema normalization, cache-marker placement, reasoning-detail replay, and token-field interpretation stay inside provider crates. Runtime modes and canonical tool definitions continue to operate on LlmRequest, LlmResponse, and ToolDefinition.
Built-in aggregator
lash-providers-builtin::register_all is the one-call aggregator the CLI and embed hosts use to register the five first-party provider factories (Anthropic, OpenAI, OpenAI-compatible, Codex, Google) with the global provider registry.

Local And Optional Dependencies

These dependencies are important operationally because they affect startup, discovery, observability, or tool availability.

MCP

attach_mcp_servers imports stdio MCP tools/list entries as mcp__server__tool dynamic tools, preserving input and output schemas where available. McpServerConfig is single-variant — stdio is the only transport; HTTP/SSE are not supported.

Tavily

Web search and URL extraction register only when LashConfig.auxiliary_secrets.tavily_api_key is present.

models.dev

CachedModelCatalog reads a file cache, can fetch https://models.dev/api.json, and has a bundled snapshot fallback.

Tracing

lash-trace defines schema version 2 JSONL records. Standard records cover turns, LLM starts/completions, tools, and summaries; extended records preserve provider/stream details. The optional otel cargo feature pulls in opentelemetry 0.31 and maps those events to spans. lash-trace-viewer is a standalone binary crate that consumes lash-trace JSONL directly, independent of lash-export.

File index

lash-file-index combines ignore walking, notify watching, and nucleo matching for async @ completion.

Store

lash/src/store.rs defines the runtime persistence contract. lash-sqlite-store is the local implementation for session heads, graph nodes, checkpoint blobs, usage deltas, tombstones, and garbage-collection metadata.

Export

lash-export renders the SQLite session plus provider trace JSONL when present, so transcript views can include prompt snapshots, context percentage, cached-token percentage, and exact token counts.
