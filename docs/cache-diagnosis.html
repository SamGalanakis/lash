cache/diagnosis

Prompt caching is a wire-level property, not a configuration promise. Diagnose it lane by lane: first prove the provider reported a regression, then inspect marker emission, prefix bytes, and session affinity in that order. The regression harness named on this page is the executable version of every check.

Detect A Regression

Graph cache reuse per logical lane, not as a fleet-wide average. A lane is the stable combination of provider path, base URL, model class, protocol, session, and agent frame.

SignalComputationInterpretation
cache-read ratiosum(cache_read_input_tokens) / sum(input_tokens + cache_read_input_tokens + cache_write_input_tokens)Compare after the lane has warmed. A sustained step down on one lane is the primary regression signal.
cache-write ratiosum(cache_write_input_tokens) / sum(total prompt buckets)Repeated writes with few reads often mean the request is cacheable but its prefix or affinity changes every call.
uncached inputinput_tokensLash normalizes this bucket after subtracting reported cache reads and writes. A rise can confirm a reuse loss.
Zero is evidence, not a verdict.

cache_read_input_tokens == 0 can mean a cold first request, a prefix below the provider's minimum cacheable size, an expired TTL, an unsupported model or endpoint, a provider that omitted usage details, or a genuine regression. Require repeated eligible calls in the same lane before alerting. Do not combine the cold call with warmed calls or average unrelated models and base URLs together.

Diagnosis Decision Tree

Keep request-side and response-side evidence separate. A zero normalized bucket cannot tell you whether the request was wrong or the provider declined a valid cache hit.

  1. Confirm the response evidence

    Inspect LlmResponse.provider_usage first. It is the raw provider usage sidecar preserved beside normalized LlmUsage. If the raw payload reports cache reads but the normalized bucket is zero, the normalizer or an alias changed; do not blame the upstream cache. The captured OpenRouter Gemini fixture is pinned by openrouter_gemini_production_usage_payload_maps_zero_cache_buckets.

  2. Is a marker emitted for this base URL and model?

    Serialize the exact path and inspect its body. Anthropic dialects use cache_control; OpenAI Responses and Codex use prompt_cache_key; unsupported paths emit nothing. cache_coverage_matrix_matches_provider_model_and_retention_dialects is the canonical provider × model × CacheRetention table. requested_breakpoints_that_the_chat_gate_strips_are_reported proves a requested marker cannot disappear without an observable drop count.

  3. Is the prefix byte-stable?

    Compare call K+1 with call K through K's cache breakpoint, after removing only the cache directive itself. Roles, block ordering, text, and string-versus-array shape remain byte-significant. Run prefix_stability_matrix_runs_consecutive_protocol_iterations plus rlm_live_bound_state_is_prefix_stable_for_every_serializer; the latter drives a real three-call RLM turn while its bound namespace evolves. The shared assertion lives in lash_llm_transport::cache_regression::assert_prefix_stability.

  4. Is session affinity stable?

    For the current OpenRouter path, inspect the request header named exactly session_id and verify that repeated calls in one session use the same value. Also check x-client-request-id remains call-specific. The openrouter-cache-affinity cutover moves affinity to a bounded body field; use the coverage expectation owned by that branch when it lands.

  5. Escalate with both sides attached

    If the request has the intended marker, a byte-stable prefix, and stable affinity, but the warmed lane still reports no reads, attach the redacted serialized request sequence plus raw provider_usage. Only then is an upstream cache miss the leading explanation.

Usage Alias Pitfalls

Gateways expose equivalent counters under different names. Diagnose from the raw sidecar and the normalizer together.

Normalized bucketKnown wire aliases
cache_read_input_tokensinput_tokens_details.cached_tokens, prompt_tokens_details.cached_tokens, top-level cached_tokens, prompt_cache_hit_tokens, Anthropic cache_read_input_tokens, Google cachedContentTokenCount
cache_write_input_tokensinput_tokens_details.cache_write_tokens, prompt_tokens_details.cache_write_tokens, top-level cache_write_tokens, Anthropic cache_creation_input_tokens
input_tokensinput_tokens, prompt_tokens, Anthropic input_tokens, Google promptTokenCount; Lash subtracts read and write buckets from totals where required.
Preserve unknown raw fields.

Cost, audio/video counters, and new gateway-specific details may not have normalized buckets. They remain available in provider_usage. Never discard that sidecar while investigating a normalization mismatch.

Five Regression Archetypes

Use this checklist during review and incident response. Each item points to the test that turns the failure class into an executable contract.

silent model gate

A base-URL/model classifier strips requested breakpoints for a model it does not recognize. Check cache_coverage_matrix_matches_provider_model_and_retention_dialects and requested_breakpoints_that_the_chat_gate_strips_are_reported.

volatile cached prefix

Iteration counters, bound variables, budgets, or other live state render before the breakpoint. Check rlm_live_bound_state_is_prefix_stable_for_every_serializer, which captures requests from live RLM execution rather than a hand-built prompt fixture.

rolling wire shape

Adding or moving a breakpoint changes an otherwise identical content block from a JSON string to an array, or back again. Check openrouter_standard_prefix_shape_is_stable_as_breakpoints_roll; it intentionally removes directives but keeps content shape significant.

inert retention

CacheRetention reaches the provider but produces no wire difference on Chat, Google direct, or Codex Long. The coverage matrix states explicitly whether each path emits cache_control, prompt_cache_key, or nothing.

hidden affinity

A provider-specific affinity field exists but its exact name, scope, or stability contract is undocumented. Verify the current session_id header and update this runbook and the matrix when the body-field cutover lands.
