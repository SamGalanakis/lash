tracing/observability

Structured runtime records emit through TraceSink. The bundled JsonlTraceSink writes one record per line; fan-out, OTel, and custom sinks are wrappers. Lashlang process tracking is a separate opt-in sink for process maps. lash-trace-viewer renders JSONL to a self-contained HTML page.

Enabling Tracing

Attach a sink at builder time. It propagates to every session from that core.

use std::sync::Arc;
use lash::{
    LashCore,
    tracing::{JsonlTraceSink, TraceLevel, TraceSink},
};

let trace_sink: Arc<dyn TraceSink> = Arc::new(JsonlTraceSink::new("./.lash-data/trace.jsonl"));

let core = LashCore::builder()
    .provider(provider)
    .model(lash::ModelSpec::from_token_limits(model.clone(), None, 200_000, None, None).expect("valid model metadata"))
    .trace_sink(Some(trace_sink))
    .trace_level(TraceLevel::Extended)
    .build()?;

TraceLevel::Standard (default): turns, tool calls, LLM start/complete, token usage. TraceLevel::Extended adds provider stream chunks, prompt-component hashes, and runtime stream deltas. Extended is verbose; use for provider debugging.

Process Tracking

Host-level Lashlang process tracking is a separate opt-in sink. It is observability only: it is not written to process_events, does not create process-registry storage, and the normal trace_sink does not receive it unless the host explicitly tees the sink itself.

let core = LashCore::builder()
    .provider(provider)
    .model(model)
    .process_tracking_jsonl_path(Some("./.lash-data/process-tracking.jsonl".into()))
    .build()?;

Tracking records use TraceEvent::ProcessTracking with typed payloads for process start/finish, node start/complete/fail, branch selection, and child process links. Every payload includes a deterministic event_key so reducers can dedupe replayed JSONL records.

The Sink Trait

One synchronous append. Called on the runtime thread; must not block. JsonlTraceSink serializes and appends under a short-held mutex.

pub trait TraceSink: Send + Sync {
    fn append(&self, record: &TraceRecord) -> Result<(), TraceSinkError>;
}

Fan out to multiple destinations by wrapping sinks:

struct FanoutTraceSink { sinks: Vec<Arc<dyn TraceSink>> }

impl TraceSink for FanoutTraceSink {
    fn append(&self, record: &TraceRecord) -> Result<(), TraceSinkError> {
        for sink in &self.sinks {
            // Treat errors per-sink; one failing destination shouldn't take the others down.
            let _ = sink.append(record);
        }
        Ok(())
    }
}

What Is Emitted

Tagged enums on TraceEvent. New variants are additive; existing fields are not renamed across schema versions.

Session lifecycle

SessionStarted, TurnStarted, TurnCompleted (with AgentFrame switch id where applicable).

Prompt

PromptBuilt — combined prompt_hash, prompt_chars, per-component fingerprints. The runtime caches rendered prompts by fingerprint; an unexpected hash change flags a drifting plugin contribution, tool list, or template entry.

LLM interaction

LlmCallStarted, LlmCallCompleted, LlmCallFailed. Provider, model, variant, timeouts, request shape.

Streaming (extended only)

ProviderStreamEvent (raw provider chunks) and RuntimeStreamEvent (post-projection SDK deltas).

Tool calls

ToolCallStarted, ToolCallCompleted. Args, output outcome (success / failure / cancelled), duration.

Usage

TokenUsage — per-turn input/output/cached/reasoning deltas. Mirrors the session usage ledger.

Protocol steps

ProtocolStep — agentic-loop iterations (RLM execute / observe / submit).

Custom

Custom { name, payload } — escape hatch for host-specific events.

The JSONL Schema

One line of UTF-8 JSON per record. TRACE_SCHEMA_VERSION = 2. Reject or warn on higher unrecognized versions.

{
  "schema_version": 2,
  "id": "6621dfa7-2cd0-4296-ac20-15c0f2d3cec1",
  "timestamp": "2026-05-11T11:42:01.234+00:00",
  "context": {
    "session_id": "chat-123",
    "turn_id": "turn-7"
  },
  "type": "tool_call_completed",
  "call_id": "call-9",
  "name": "read_file",
  "args": { "...": "..." },
  "output": { "outcome": { "status": "success", "payload": { "...": "..." } } },
  "duration_ms": 8
}

Record type: lash_trace::TraceRecord. Event variants: lash_trace::TraceEvent. Both Deserialize; parse directly with serde_json::from_str::<TraceRecord>().

Viewing A Trace

Renders JSONL to a self-contained HTML timeline with nested LLM calls, tool calls, and streaming chunks inline. Valid-JSON records of an unrecognized schema are preserved as raw text; a line that is not valid JSON aborts the load.

$ cargo run -p lash-trace-viewer -- ./.lash-data/trace.jsonl
# Writes ./.lash-data/trace.html next to the input

$ cargo run -p lash-trace-viewer -- trace.jsonl \
      --out report.html \
      --title "Session 2026-05-11"
# Custom output path and title.

Workspace-only; no separate release. Run with cargo run -p lash-trace-viewer from the workspace root.

OpenTelemetry Export

Optional otel-trace cargo feature converts events to OTel spans for export to Jaeger, Honeycomb, Datadog, or any OTLP backend. Off by default. The sink lives in lash-trace and is re-exported by lash-core under the feature.

# In your downstream Cargo.toml — pull in lash-core with the otel-trace
# feature alongside the lash-runtime facade.
lash-runtime = "=0.1.0-alpha.17"
lash-core    = { version = "=0.1.0-alpha.17", features = ["otel-trace"] }

lash_core::OtelTraceSink wraps an opentelemetry tracer and converts every event to a span with payload attributes. Use in place of (or alongside) JsonlTraceSink:

use lash::{LashCore, tracing::{TraceLevel, TraceSink}};
use lash_core::OtelTraceSink;

let sink: Arc<dyn TraceSink> = Arc::new(OtelTraceSink::new(tracer));
let core = LashCore::builder()
    .trace_sink(Some(sink))
    .trace_level(TraceLevel::Extended)
    .build()?;
