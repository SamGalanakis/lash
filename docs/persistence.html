session/persistence

Runtime state is persisted through RuntimePersistence. The default impl is lash-sqlite-store. Install a SessionStoreFactory on the core, or pass a store per session.

The Contract

Runtime-facing trait. Most hosts use lash-sqlite-store rather than implementing it.

#[async_trait]
pub trait RuntimePersistence: Send + Sync {
    async fn load_session(&self, scope: SessionReadScope)
        -> Result<Option<PersistedSessionRead>, StoreError>;
    async fn load_node(&self, node_id: &str)
        -> Result<Option<SessionNodeRecord>, StoreError>;
    async fn commit_runtime_state(&self, commit: RuntimeCommit)
        -> Result<RuntimeCommitResult, StoreError>;
    async fn save_session_meta(&self, meta: SessionMeta) -> Result<(), StoreError>;
    async fn load_session_meta(&self) -> Result<Option<SessionMeta>, StoreError>;
    async fn tombstone_nodes(&self, ids: &[String]) -> Result<(), StoreError>;
    async fn vacuum(&self) -> Result<VacuumReport, StoreError>;
    async fn gc_unreachable(&self) -> Result<GcReport, StoreError>;
}

SessionStoreFactory produces one store per session, keyed by session id, parent id, and policy:

pub trait SessionStoreFactory: Send + Sync {
    fn create_store(
        &self,
        request: &SessionStoreCreateRequest,
    ) -> Result<Arc<dyn RuntimePersistence>, String>;
}

Concurrent Writes And CAS

Each RuntimeCommit carries expected_head_revision: Option<u64>. The store checks it against the row's current revision in the same SQL transaction; on mismatch the transaction rolls back. No retry, no partial state.

At most one turn per session commits at a time. Concurrent turns from two processes (or two tasks) on the same session id race for the head; the loser surfaces store_commit_failed.

match session.turn(input).run().await {
    Ok(turn) => persist(turn),
    Err(err) if err.code == "store_commit_failed" => {
        // Another writer raced us. Re-open the session to read the new
        // head, then decide whether to retry, surface to the user, or
        // merge intent.
        let session = core.session(chat_id).open().await?;
        retry_or_surface(err, session);
    }
    Err(other) => bail!(other),
}

Conflicts surface as a turn-level Err with RuntimeError { code: "store_commit_failed", .. } — not a TurnStop variant. The next read is identical to the read before the failed call. Hosts expecting concurrent access (multi-tab UIs, agent fleets) should serialize at their own layer or re-open against the refreshed head.

In-memory sessions (no store_factory) enforce the same CAS against an in-process counter.

For per-effect durability under an external workflow runtime, install a custom RuntimeEffectController; see Architecture → Durability.

Wiring It Up

Install one factory on the core; every session inherits it.

use std::sync::Arc;
use lash::{LashCore, ModeId, ModePreset};
use lash_sqlite_store::SqliteSessionStoreFactory;

let store_factory = Arc::new(SqliteSessionStoreFactory::new("./.lash-data"));

let core = LashCore::builder()
    .install_mode(ModePreset::rlm())
    .default_mode(ModeId::rlm())
    .provider(provider)
    .model(model, None)
    .max_context_tokens(200_000)
    .store_factory(store_factory)
    .build()?;

Per-session override (e.g. tests with an isolated store):

let session = core
    .session(chat_id)
    .store(Arc::new(my_custom_persistence))
    .open()
    .await?;

Without a store_factory, sessions run in-memory: turns succeed but nothing is written or resumable. There is no separate in-memory impl — absence of a factory is the mechanism.

What Gets Persisted

Per-session SQLite contents:

Graph nodes

User inputs, assistant responses, tool calls, mode events, prompt snapshots in graph_nodes, with a tombstoned flag for soft deletion.

Session head

Singleton row with head pointer and revision. Revision drives CAS.

Checkpoint blobs

Compressed snapshots of tool, plugin, execution, and prompt state, content-hashed and deduplicated across resumes.

Usage deltas

Per-turn input_tokens, output_tokens, cached_input_tokens, reasoning_tokens. Aggregated into the session usage ledger on SessionReadView.

Metadata

Session id, name, model, cwd, parent id, created-at. What the resume picker shows.

Attachments

Image / file blobs referenced from graph nodes via BlobRef. Shared with traces and exports.

Garbage Collection

Two reclamation ops on RuntimePersistence. Safe to run from a background task or CLI command.

gc_unreachable()

Walks reachable blobs from head and active checkpoints, marks orphans for deletion, returns a GcReport. Run after pruning a branch.

vacuum()

Removes tombstoned graph-node rows already detached by gc, returns a VacuumReport. Run after gc_unreachable to reclaim file size.

End-To-End Example

The agent-service example: one SQLite factory, sessions keyed by chat id, app database alongside.

// One factory at boot, shared across every chat.
let store_factory = Arc::new(SqliteSessionStoreFactory::new(
    data_dir.join("lash-sessions"),
));

let core = LashCore::builder()
    .install_mode(ModePreset::rlm())
    .default_mode(ModeId::rlm())
    .provider(provider)
    .model(model.clone(), Some(model_variant.clone()))
    .max_context_tokens(200_000)
    .store_factory(store_factory)
    .build()?;

// Per request: open a session keyed by the app's chat id.
let session = core.session(chat_id).rlm().open().await?;
