session/persistence

Runtime state is persisted through the RuntimePersistence contract. There is no default store: install a SessionStoreFactory on the core, or pass a store per session, otherwise sessions stay in-process. lash-turso-store is the provided durable implementation; lash-core ships an in-memory one.

What Gets Persisted

The state every store holds per session, as the contract guarantees (column and table names below are the Turso store's layout):

Graph nodes

User inputs, assistant responses, tool calls, protocol events, prompt snapshots in graph_nodes, with a tombstoned flag for soft deletion.

Session head

Singleton row with head pointer, revision, current AgentFrame id, and frame records. Revision drives CAS.

Checkpoint blobs

Compressed snapshots of committed tool, plugin, current-frame execution, and prompt state, content-hashed and deduplicated across store reopens.

Queued work

Lease-claimed batches for queued turn input, process wakes, host events, and timers. Completed claim ids are cleared atomically with the runtime commit that consumed them.

Turn commit stamps

Idempotency rows keyed by session and turn id. Identical final commit retries return the recorded result; changed hashes are rejected.

Usage deltas

Per-turn input_tokens, output_tokens, cached_input_tokens, reasoning_tokens. Aggregated into the session usage ledger on SessionReadView.

Metadata

Session id, name, model, cwd, relation (Root or Child), and created-at. What the resume picker shows.

Attachments

Image / file blobs referenced from graph nodes via BlobRef. Shared with traces and exports.

The Contract

Runtime-facing trait. The contract covers session graph/head commits, queued work, metadata, garbage collection, final turn commit idempotency, and the attachment write-ahead manifest inherited through AttachmentManifest. Stores do not execute or replay in-flight effects; durable workflow hosts own that through EffectHost history and timers. Most hosts wire a provided implementation (TursoSessionStoreFactory for durability, InMemorySessionStoreFactory for ephemeral runs) rather than implementing the trait themselves.

#[async_trait]
pub trait RuntimePersistence: AttachmentManifest + Send + Sync {
    fn durability_tier(&self) -> DurabilityTier { DurabilityTier::Inline }
    async fn load_session(&self, scope: SessionReadScope)
        -> Result<Option<PersistedSessionRead>, StoreError>;
    async fn load_node(&self, node_id: &str)
        -> Result<Option<SessionNodeRecord>, StoreError>;
    async fn commit_runtime_state(&self, commit: RuntimeCommit)
        -> Result<RuntimeCommitResult, StoreError>;
    async fn enqueue_queued_work(&self, batch: QueuedWorkBatchDraft)
        -> Result<QueuedWorkBatch, StoreError>;
    async fn claim_ready_queued_work(
        &self,
        session_id: &str,
        owner_id: &str,
        boundary: QueuedWorkClaimBoundary,
        lease_ttl_ms: u64,
        max_batches: usize,
    ) -> Result<Option<QueuedWorkClaim>, StoreError>;
    async fn renew_queued_work_claim(
        &self,
        claim: &QueuedWorkClaim,
        lease_ttl_ms: u64,
    ) -> Result<QueuedWorkClaim, StoreError>;
    async fn abandon_queued_work_claim(&self, claim: &QueuedWorkClaim)
        -> Result<(), StoreError>;
    async fn list_queued_work(&self, session_id: &str)
        -> Result<Vec<QueuedWorkBatch>, StoreError>;
    async fn save_session_meta(&self, meta: SessionMeta) -> Result<(), StoreError>;
    async fn load_session_meta(&self) -> Result<Option<SessionMeta>, StoreError>;
    async fn tombstone_nodes(&self, ids: &[String]) -> Result<(), StoreError>;
    async fn vacuum(&self) -> Result<VacuumReport, StoreError>;
    async fn gc_unreachable(&self) -> Result<GcReport, StoreError>;
}

pub struct RuntimeCommit {
    pub session_id: String,
    pub expected_head_revision: Option<u64>,
    pub config: PersistedSessionConfig,
    pub agent_frames: Vec<AgentFrameRecord>,
    pub current_agent_frame_id: AgentFrameId,
    pub graph: GraphCommitDelta,
    pub checkpoint: HydratedSessionCheckpoint,
    pub usage_deltas: Vec<TokenLedgerEntry>,
    pub turn_commit: Option<RuntimeTurnCommitStamp>,
    pub completed_queue_claims: Vec<QueuedWorkCompletion>,
    pub committed_attachment_ids: Vec<AttachmentId>,
}

SessionStoreFactory produces one store per session, keyed by session id, relation, and policy:

pub trait SessionStoreFactory: Send + Sync {
    fn create_store(
        &self,
        request: &SessionStoreCreateRequest,
    ) -> Result<Arc<dyn RuntimePersistence>, String>;
    fn delete_session(&self, session_id: &str) -> Result<(), String>;
}

Concurrent Writes And CAS

Each RuntimeCommit carries expected_head_revision: Option<u64>. The store checks it against the row's current revision in the same SQL transaction; on mismatch the transaction rolls back. No retry, no partial state.

At most one turn per session commits at a time. Concurrent turns from two processes (or two tasks) on the same session id race for the head; the loser surfaces store_commit_failed.

match session.turn(input).run().await {
    Ok(turn) => persist(turn),
    Err(err) if err.code == "store_commit_failed" => {
        // Another writer raced us. Re-open the session to read the new
        // head, then decide whether to retry, surface to the user, or
        // merge intent.
        let session = core.session(chat_id).open().await?;
        retry_or_surface(err, session);
    }
    Err(other) => bail!(other),
}

Conflicts surface as a turn-level Err with RuntimeError { code: "store_commit_failed", .. } — not a TurnStop variant. The next read is identical to the read before the failed call. Hosts expecting concurrent access (multi-tab UIs, agent fleets) should serialize at their own layer or re-open against the refreshed head.

In-memory sessions (no store_factory) enforce the same CAS against an in-process counter.

For per-effect durability under an external workflow runtime, install a durable EffectHost. Local stores reopen the last committed state after a crash. Restate-style hosts use workflow history and timers to replay in-flight effects, then rely on RuntimeTurnCommitStamp so a repeated final commit is idempotent. See Architecture → Durability.

Wiring It Up

Install one session-store factory plus the explicit runtime facets on the core; every session inherits them.

use std::sync::Arc;
use lash::{LashCore, ModeId, ModePreset};
use lash_turso_store::{TursoSessionStoreFactory, Store};

let data_dir = std::path::PathBuf::from("./.lash-data");
let store_factory = Arc::new(TursoSessionStoreFactory::new(data_dir.join("sessions")));
let artifact_store = Arc::new(Store::open(&data_dir.join("artifacts.db"))?);

let core = LashCore::builder()
    .install_mode(ModePreset::rlm())
    .default_mode(ModeId::rlm())
    .provider(provider)
    .model(lash::ModelSpec::from_token_limits(model.clone(), None, 200_000, None, None).expect("valid model metadata"))
    .store_factory(store_factory)
    .effect_host(Arc::new(lash::durability::InlineEffectHost::default()))
    .lashlang_artifact_store(artifact_store)
    .attachment_store(Arc::new(lash::persistence::FileAttachmentStore::new(data_dir.join("attachments"))))
    .build()?;

Per-session override (e.g. tests with an isolated store):

let session = core
    .session(chat_id)
    .store(Arc::new(my_custom_persistence))
    .open()
    .await?;

Without a store_factory, session state is in-process only: turns succeed but nothing is written or resumable. You still provide explicit in-memory effect, artifact, and attachment facets with InlineEffectHost, InMemoryLashlangArtifactStore, and InMemoryAttachmentStore.

Garbage Collection

Two reclamation ops on RuntimePersistence. Safe to run from a process or CLI command.

gc_unreachable()

Walks reachable blobs from head and active checkpoints, marks orphans for deletion, returns a GcReport. Run after pruning a branch.

vacuum()

Removes tombstoned graph-node rows already detached by gc, returns a VacuumReport. Run after gc_unreachable to reclaim file size.

End-To-End Example

The agent-service example: one session-store factory (Turso here), sessions keyed by chat id, app database alongside.

// One factory at boot, shared across every chat.
let store_factory = Arc::new(TursoSessionStoreFactory::new(
    data_dir.join("lash-sessions"),
));
let artifact_store = Arc::new(lash_turso_store::Store::open(
    &data_dir.join("lash-artifacts.db"),
)?);

let core = LashCore::builder()
    .install_mode(ModePreset::rlm())
    .default_mode(ModeId::rlm())
    .provider(provider)
    .model(lash::ModelSpec::from_token_limits(model.clone(), Some(model_variant.clone()), 200_000, None, None).expect("valid model metadata"))
    .store_factory(store_factory)
    .effect_host(Arc::new(lash::durability::InlineEffectHost::default()))
    .lashlang_artifact_store(artifact_store)
    .attachment_store(Arc::new(lash::persistence::FileAttachmentStore::new(
        data_dir.join("attachments"),
    )))
    .build()?;

// Per request: open a session keyed by the app's chat id.
let session = core.session(chat_id).rlm().open().await?;
