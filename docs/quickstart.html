quick/start

The shortest path to a working Lash turn. Add the crate, pick a provider, open a session, run one turn. Tools, modes, plugins, persistence, and tracing all layer on after that.

§ 01

depend on it

add the crates

The lash-runtime crate is the user-facing facade (published on crates.io as lash-runtime because lash is owned by another project; the library is still imported as use lash::...). Pull in whatever provider crate you actually use and pass a configured ProviderHandle when opening sessions.

During the alpha series, versions carry an -alpha.N suffix, so the dep needs the explicit pre-release tag.

[dependencies]
lash-runtime         = "=0.1.0-alpha.19"
lash-provider-openai = "=0.1.0-alpha.19"
anyhow               = "1"
tokio                = { version = "1", features = ["full"] }

For ordinary projects the bundled facade and provider crates cover the common ground. Drop down to lash-core only when you need runtime-host plumbing — custom PluginHost instances or an explicitly installed deployment-level lash::persistence::ProcessRegistry such as lash-sqlite-store::SqliteProcessRegistry for background process handles.

§ 02

read the shape

core concepts

The minimal example below names LashCore, LashSession, TurnInput, TurnBuilder, and ProviderHandle. SessionSpec and PluginStack are the next two types you'll meet in the Lash API guide. Skim them before reading the code — the rest follows from how they compose.

LashCore
Cloneable shared configuration: provider, model, installed modes, tools, plugins. One per app, cloned freely.
SessionSpec
Reusable session configuration overlay for provider, model, mode, prompt layer, max context tokens, and max turns. The ergonomic builder methods update the same spec.
PluginStack
Mutable list of plugin factories. LashCore::standard() and LashCore::rlm() start with the runtime defaults; LashCore::builder() is explicit and starts empty.
LashSession
One logical conversation or task, opened from a LashCore by a stable session id.
TurnInput
Semantic input to a turn — text plus image references. Use TurnInput::text(...) for the common case.
TurnBuilder
Per-turn configuration returned by session.turn(input). Call .run() for a collected result, .stream(&sink) for live events.
ProviderHandle
A live transport adapter supplied by the host. Session persistence stores only the provider id and rebinds it to the host's active handle when a session opens.

§ 03

run one turn

hello, turn

The smallest meaningful program: build a core, open a session, run one turn, print the settled assistant prose.

use lash::{provider::ProviderHandle, LashCore, TurnInput};
use lash_provider_openai::{OPENROUTER_BASE_URL, OpenAiCompatibleProvider};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    // build a provider handle. substitute your own creds + base URL.
    let api_key = std::env::var("OPENROUTER_API_KEY")?;
    let provider = ProviderHandle::new(
        OpenAiCompatibleProvider::new(api_key, OPENROUTER_BASE_URL).into_components(),
    );

    // one LashCore per app, cloned freely.
    let core = LashCore::standard()
        .provider(provider)
        .model(lash::ModelSpec::from_token_limits("anthropic/claude-sonnet-4.6", None, 200_000, None, None).expect("valid model metadata"))
        .build()?;

    // one session per chat / task.
    let session = core.session("hello-1").open().await?;

    // run one turn; read settled prose from the terminal result.
    let result = session
        .turn(TurnInput::text("Say hi in one short sentence."))
        .run()
        .await?;

    let prose = result.assistant_message().unwrap_or_default();
    println!("{prose}");
    Ok(())
}

That's the whole flow: LashCore for shared configuration, LashSession for one logical conversation, TurnBuilder::run() for a single collected turn. Nothing more is required.

§ 04

what comes next

what's not here yet

The example above has no persistence, no host tools, only the default runtime plugin stack, no tracing, and no live streaming. Each is a small additional step — pick what you need and read on.

live streaming

Use TurnBuilder::stream(&sink) instead of run() to emit TurnActivity items as the model produces them. The Lash API · turns guide covers the full event taxonomy.

host tools

Implement ToolProvider, hand it to .tools(Arc::new(MyTools)) on the builder. See the Plugins guide.

persistence

Pass a SessionStoreFactory to .store_factory(...) so sessions survive process restarts. See the Persistence guide.

tracing

Attach a TraceSink via .trace_sink(Some(Arc::new(sink))) for JSONL traces of every turn. See the Tracing guide.

rlm mode

Install ModePreset::rlm() for Lashlang-driven agentic turns with structured submit / AgentFrame switch semantics. See RLM and Submit.

mcp servers

Wire up MCP via McpPluginFactory::new(servers).await? and pass it to .plugin(...). See MCP Servers in the Lash API guide.
