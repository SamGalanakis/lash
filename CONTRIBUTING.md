# Contributing to lash

Feature requests and bug reports are welcome — open an
[issue](https://github.com/SamGalanakis/lash/issues).

At this alpha stage, **detailed write-ups help more than drive-by PRs**. The
internals are still moving fast and code can easily land in the wrong direction,
so a clear report — what you tried, what you expected, and what actually
happened — is far more useful than a patch against a surface that may be about to
change. If you do want to discuss an implementation, open an issue first so we
can agree on the shape before anyone writes code.

To understand how the runtime fits together, start at <https://lash.run/> — the
architecture chapters cover the crate layout, the turn/effect boundary, and the
plugin model.
